# Q002 — Renderer Frame 전달과 RHI Word Stream 실행 경로

## 질문 원문
Renderer의 프레임 생산 책임, 불변 Snapshot 전달, Render Thread의 RHI Command 생성 위치와 UE식 공개 진입점 이름을 확정하고 #2_5 구현 기준을 정리한다.

## 전제 / 범위
- Runtime/World는 월드 상태를 확정하고 불변 RenderFrame Snapshot을 생산한다.
- Renderer Thread는 Snapshot을 소비해 렌더링 파이프라인과 RHI Command Stream을 생성한다.
- Renderer 구현 DLL은 RHI Command Stream을 플랫폼 API로 번역·실행한다.
- #2_5의 가시 결과는 ClearRenderTarget + Present이다.
- Scene/View/Resource Handle/Resize/Capability/PipelineDescriptor는 제외한다.
