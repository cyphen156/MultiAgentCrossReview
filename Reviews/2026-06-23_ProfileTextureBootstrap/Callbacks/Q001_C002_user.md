# Q001_C002 - Debug fixture scope for #2_6 display

## 사용자 판정

C는 정식 릴리즈에서 당연히 수행되는 기능일거야 정상적으로 커맨드큐에 태워서 렌더 타겟 전달하고 그래픽 메모리 사용하기 시작하면 근데 그걸 하려면 리소스 매니지먼트가 들어오고 텍스쳐나 매시에 대한 해석방법이 존재하는 모델 뷰어 체계가 갖춰져야함 

지금 그냥 냅따 파일하나 올려서 이거 무조건 JPG니까 텍스쳐로 렌더링 해라 라는 명령이니 디버그 테스트만 수행

## 반영 조건

- 정식 릴리즈/런타임 방향에서는 command queue, render target 전달, GPU memory, resource management, texture/mesh 해석, model viewer 체계가 필요하다.
- #2_6의 현재 표시 목표는 정식 runtime 기능이 아니라 debug test fixture/demo로 한정한다.
- `RendererTest.cpp`가 "이 파일은 JPG이므로 texture로 렌더링한다"는 테스트 데이터를 적재하고 실행하는 위치가 된다.
- Release 동작 보장은 이번 #2_6의 합격 조건이 아니다.
- 이후 runtime 승격은 ResourceManager / ModelViewer / asset interpretation 구조가 생기는 별도 단계로 분리한다.
