# Q001 — 브랜치 2 Renderer Command Stream(#2_5) 구현안 검토

## 질문 원문
최신 #2_4 기준으로 작성한 #2_5 구현안. 검토 미러는 읽기 전용이므로 직접 적용하지 않았다.

핵심 흐름:

	Engine Thread
		Renderer::Render()
			→ 64bit Word Stream 생성
			→ Command Buffer 단위 Queue 제출

	Render Thread
		Queue에서 Buffer 획득
			→ ExecuteCommandList() 1회

	DX11 DLL
		Word Stream 해석
			→ ClearRenderTargetView
			→ Present

신규/변경 대상:
- RenderCommand.h (Word/Header/Payload ABI, static_assert)
- RenderCommandBuffer (.h/.cpp) — 64bit Word 연속 저장 + 재사용
- RenderCommandQueue (.h/.cpp) — 단일 슬롯 backpressure handoff
- RendererModule.h — ABI generation 4, create/destroy/executeCommandList 함수포인터
- Renderer (.h/.cpp) — Render Thread 수명, Render()/Run()/Shutdown()
- CyphenEngine.cpp — Engine Loop에서 Renderer::Render() 생산
- Dx11Renderer (.h/.cpp) — Device/SwapChain/RTV + Command Parser
- Dx11RendererModule.cpp — ExecuteCommandList export

"검토해봐"

## 전제 / 범위
- 대상 미러는 읽기 전용. 본 검토는 설계·정확성 판단이며 원본 코드 적용은 사용자가 수행한다.
- #2_4까지의 결론(select-before-load, ModuleManager=집행, Renderer=실행 상태 SSOT, EngineContext=공통 환경, device 스레드 친화)을 전제로 한다.
- 제외: Resize / Alt+Enter / WM_SIZE, 범용 serializer/container, 중첩 stream/압축/가변 문자열/범용 Payload API.
- #2_5 명령 집합은 ClearRenderTarget, Present 둘로 한정한다.
